# app Architecture

## Classification

`app` is the RustZen product site. It publishes brand, product, and blog content for RustZen macOS tools at `app.rustzen.dev`. It is not a Rust backend, Web admin dashboard, Tauri client, or release bundle project.

## Stack

- Astro static output (`astro.config.mjs` sets `output: 'static'`).
- MDX via `@astrojs/mdx`.
- Astro content collections via `src/content.config.ts`.
- TypeScript data modules for product metadata.
- Vercel static deployment using `vercel.json`.

## Source Layout

| Path | Role | Fact boundary |
| --- | --- | --- |
| `src/pages/index.astro` | Home page and latest blog strip. | tracked source |
| `src/pages/[code].astro` | Static product pages generated from `src/data/tools.ts`. | tracked source |
| `src/pages/checkout/success.astro` | Static checkout return page for completed Creem purchases. | source candidate until tracked |
| `src/pages/blog/index.astro` | Blog index. | tracked source |
| `src/pages/blog/[...slug].astro` | Blog article routes. | tracked source |
| `src/layouts/SiteLayout.astro` | Shared document shell, navigation, footer, metadata baseline. | tracked source |
| `src/data/tools.ts` | Product names, routes, status, descriptions, highlights, and proof copy. | tracked source |
| `src/content.config.ts` | Blog collection schema and loader. | tracked source |
| `src/content/posts/*.mdx` | Blog article content. | tracked source or modified tracked during active editing |
| `src/assets/blog/*` | Blog-owned source assets. | tracked source |
| `public/*` | Static public assets, icons, fonts, and product media. | tracked or untracked until added |
| `docs/*` | Governance and content records. | tracked or modified tracked |

## Routes

- `/`: RustZen product site home.
- `/zen-clear`: Zen Clear product page.
- `/clipboard`: Clipboard product page.
- `/checkout/success`: static return page after a completed Creem checkout.
- `/blog`: blog index.
- `/blog/<slug>`: MDX blog article route.

Product routes are generated by `getStaticPaths()` in `src/pages/[code].astro` from `src/data/tools.ts`; route changes should update both data and any affected navigation/copy.

Zen Clear purchase copy is owned by `src/data/tools.ts`. The current public
offer is Pro as an annual Creem subscription at `$10/year`, linking through
`rustzen-cloud` at `/api/billing/checkout?product=rustzen-clear&source=site`.

## Content Collections

The `posts` collection is loaded from `src/content/posts/**/*.{md,mdx}`. Frontmatter must match the schema in `src/content.config.ts`:

- `title`
- `description`
- `pubDate`
- `lang`
- `translationKey`
- `source`
- `tags`
- optional `author`
- optional `cover`
- optional `heroImage`

## Boundaries

The site has no frontend/backend API contract in this repository. Pages are
pre-rendered from `src/pages`, `src/data/tools.ts`, and the `posts` content
collection during the Astro static build.

Do not introduce Rust service deployment assumptions here:

- no `/opt/rustzen-app`;
- no systemd unit;
- no Docker release rule;
- no `target/rustzen-app`;
- no `apps/server` or `apps/web` migration;
- no Tauri updater/signing/capabilities rules.

Generated and local-only output must not become source truth:

- `.vercel/` is ignored local deployment link metadata;
- `.astro/`, `.next/`, `out/`, and `dist/` are ignored generated output or stale framework output;
- source, docs, and public assets must be reviewed through Git.
